# AiriLab Skill 优化报告

## 优化日期
2026-04-08

## 问题诊断

### 原问题
1. **Agent 不调用 skill** - description 太短，缺少触发关键词
2. **不按脚本执行** - 缺少明确的运行方式和参数说明
3. **意图匹配弱** - 没有明确的触发条件说明
4. **配置不透明** - 用户不知道需要什么前置条件

### 根本原因
对比其他正常工作的 skill（如 culture-analyzer、site-analyzer），发现原 SKILL.md 缺少：
- 详细的触发关键词列表
- metadata 元数据（emoji、requires）
- 明确的触发条件章节
- 前置条件说明
- 输出结构说明

## 优化内容

### 1. 增强 SKILL.md

#### 1.1 更新 description
**优化前：**
```
统一调用 AiriLab 图像生成能力（MJ/创意放大/氛围转换），采用同步等待模式
```

**优化后：**
```
调用 AiriLab AI 图像生成平台，支持建筑/室内/景观/城市规划的 AI 渲染。当用户需要「生成建筑效果图」「AI 绘图」「创意渲染」「图片放大」「氛围转换」「参考图生成」「MJ 渲染」「生成室内设计图」「生成景观图」「生成城市设计图」「生成效果图」「AI 出图」「快速渲染」「概念图生成」时使用此技能。支持文生图、图生图、参考图控制。
```

#### 1.2 添加 metadata
```yaml
metadata: { "openclaw": { "emoji": "🎨", "requires": { "bins": ["python3"], "pip": ["requests"] } } }
```

#### 1.3 新增章节
- **前置条件** - 说明需要的配置和依赖
- **工作流支持** - 表格形式清晰展示三种工作流
- **触发条件** - 详细说明什么情况下应该触发
  - 模式 1: 文生图（MJ 创意渲染）
  - 模式 2: 图片放大
  - 模式 3: 氛围转换
  - 模糊触发（智能识别）
- **运行方式** - 两种调用方法
- **输出结构** - JSON 格式说明
- **响应格式** - 成功/失败响应示例
- **常见错误处理** - 错误原因和解决方案表格
- **配置检查** - 快速诊断命令

## 使用建议

### 对 Agent
1. 当用户提到任何触发关键词时，应主动调用此 skill
2. 优先使用 `core/api.py` 脚本执行
3. 同步等待结果（1-3 分钟）
4. 返回结果时展示所有输出图片 URL

### 对用户
1. 首次使用前运行配置检查：
   ```bash
   python3 scripts/check_config.py
   ```
2. 直接使用自然语言描述需求，例如：
   - "帮我生成一张现代建筑的效果图"
   - "把这张图放大"
   - "把这张图转换成夜晚氛围"
